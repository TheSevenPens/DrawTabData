# DATA LAYOUT

JSON files are organized by entity type, mostly partitioned by brand.

	/data
		/brands
			brands.json
		/tablets
			GAOMON-tablets.json
			HUION-tablets.json
			UGEE-tablets.json
			WACOM-tablets.json
			XENCELABS-tablets.json
			XPPEN-tablets.json
		/pens
			HUION-pens.json
			SAMSUNG-pens.json
			WACOM-pens.json
			XENCELABS-pens.json
			XPPEN-pens.json
		/pen-families
			WACOM-pen-families.json
		/tablet-families
			WACOM-tablet-families.json
		/pen-compat
			HUION-pen-compat.json
			SAMSUNG-pen-compat.json
			WACOM-pen-compat.json
			XENCELABS-pen-compat.json
			XPPEN-pen-compat.json
		/drivers
			WACOM-drivers.json
		/pressure-response
			HUION-pressure-response.json
			SAMSUNG-pressure-response.json
			WACOM-pressure-response.json
			XENCELABS-pressure-response.json
			XPPEN-pressure-response.json
		/inventory
			sevenpens-pens.json
			sevenpens-tablets.json
		/reference
			defect-kinds.json
			iso-paper-sizes.json
			us-paper-sizes.json

Each file wraps its records in a top-level key:
- brands: { "Brands": [...] }
- tablets: { "DrawingTablets": [...] }
- pens: { "Pens": [...] }
- pen-families: { "PenFamilies": [...] }
- tablet-families: { "TabletFamilies": [...] }
- pen-compat: { "PenCompat": [...] } (grouped: PenId -> TabletIds[])
- drivers: { "Drivers": [...] }
- pressure-response: { "PressureResponse": [...] }
- inventory pens: { "InventoryPens": [...] }
- inventory tablets: { "InventoryTablets": [...] }
- defect kinds: { "DefectKinds": [...] }
- iso paper sizes: { "ISOPaperSizes": [...] }
- us paper sizes: { "USPaperSizes": [...] }

File naming:
- Most entities: BRAND-entitytype.json (e.g. WACOM-tablets.json)
- Brands: brands.json (single file, not partitioned)
- Inventory: USERNAME-entitytype.json (e.g. sevenpens-pens.json)
- Reference: plain name, not partitioned (e.g. defect-kinds.json)

See docs/DEFECTS.md for the Defects field on inventory records
and the defect-kinds vocabulary.
