# Introduction

The DrawTabData project unifies JSON datasets related to Drawing Tablets.

Entities include:
- Brands (11)
- Tablets (290 across 7 brands)
- Pens (104 across 5 brands)
- Pen Families (10 Wacom)
- Tablet Families (18 Wacom)
- Pen Compatibility (100 groups across 5 brands)
- Drivers (246 Wacom)
- Pressure Response (124 sessions across 5 brands)
- Inventory (263 items for user sevenpens)

# The brand dimension

Brands currently in the system:
- APPLE
- ASUS
- GAOMON
- HUION
- SAMSUNG
- STAEDTLER
- UGEE
- VEIKK
- WACOM
- XENCELABS
- XPPEN

Datasets are partitioned as files by brand (e.g. WACOM-tablets.json).
Inventory is partitioned by user (e.g. sevenpens-pens.json).
Brands have display names (e.g. XPPEN -> "XP-Pen").

# EntityId convention

Every entity has an EntityId in the format BRAND.ENTITYTYPE.ID:
- WACOM.TABLET.PTK870
- WACOM.PEN.KP501E
- WACOM.DRIVER.6.3.45-2_WINDOWS
- WACOM.PENFAMILY.WacomKPGEN3
- WACOM.TABLETFAMILY.WacomIntuosProGen8
- WACOM.BRAND.WACOM

# Libraries

This repo includes TypeScript libraries for consumers:

- lib/schemas.ts — Valibot schemas (single source of truth for shape + types)
- lib/drawtab-loader.ts — URL-based loaders (types re-exported from schemas)
- lib/drawtab-loader-node.ts — filesystem-based loaders for Node.js
- lib/loader-shared.ts — shared brand lists, expandPenCompat, parseStringArray
- lib/drawtab-all.ts / lib/drawtab-all-node.ts — load everything in one call
- lib/compat-helpers.ts — compatibility map builders + findSimilarTablets
- lib/pipeline/ — generic query engine (filter, sort, select, take)
- lib/entities/ — field definitions per entity type (metadata, computed fields)
- lib/units.ts — metric/imperial unit conversion
- lib/data-quality.ts — schema-driven validation across all entities (npm run data-quality)
- scripts/generate-version.ts — regenerates data/version.json (npm run version-info)
- data/version.json — schemaVersion, date-version, commit SHA, and per-entity counts

# How consumers use the data

- Include this repo as a git submodule
- Clone the repo
- Copy data/ and lib/ into your project
- Fetch data at runtime from GitHub Pages or raw.githubusercontent
- The DrawTabDataExplorer repo is the primary consumer (SvelteKit UI)
